# <a name="definitions"></a>定義

本檔提供此空間中特定單字和片語的定義，並在整個檔集中當做參考使用。

## <a name="command-line-applications"></a>命令列應用程式

命令列應用程式（有時稱為「主控台應用程式」和/或稱為「用戶端」的主控台子系統）是主要針對文字或字元資訊資料流程運作的程式。 它們通常不包含自己的使用者介面專案，而且會將輸出/顯示和輸入/互動角色委派給裝載應用程式。 命令列應用程式會在其標準輸入控制碼上接收文字資料流程 `STDIN` ，這表示使用者的鍵盤輸入、處理該資訊，然後以標準輸出上的文字資料流程回應， `STDOUT` 以顯示回使用者的監視器。 當然，對於其他輸入裝置和遠端案例而言，這會隨著時間而進化，但相同的基本原理仍然相同：命令列用戶端會在文字上操作，而其他人則會管理顯示/輸入。

## <a name="standard-handles"></a>標準控制碼

標準控點是一系列、 `STDIN` 、 `STDOUT` 和 `STDERR` ，在啟動時導入為進程空間的一部分。 它們代表在中要接受和送回的資訊位置， (包括將錯誤回報) 的特殊位置。 如果是命令列應用程式，這些應用程式必須一律存在於應用程式啟動時。 它們會自動從父系繼承、由父系明確設定，或者如果未指定/允許，則由作業系統自動建立。 若為傳統 Windows 應用程式，這些應用程式在啟動時可能會空白。 不過，它們可以隱含或明確地繼承自父系，或在執行時間由應用程式本身進行配置、附加和釋放。

標準控點不代表特定類型的已連接裝置。 不過，在命令列應用程式的情況下，裝置最常是主控台裝置、shell) 中的重新導向 (的檔案，或是從將一個公用程式的輸出連接到下一個) 之輸入的命令介面 (的管道。 它也可能是通訊端或任何其他類型的裝置。

## <a name="ttypty"></a>TTY/PTY

在非 Windows 平臺上，TTY 和 PTY 裝置分別代表真正的實體裝置或軟體建立的虛擬裝置，與 Windows 主控台會話的概念相同：命令列用戶端應用程式與伺服器主機互動性應用程式之間通訊的通道，或實體鍵盤/顯示裝置可以交換以文字為基礎的資訊。

## <a name="clients-and-servers"></a>用戶端和伺服器

在此空間中，我們將「用戶端」視為可執行處理資訊和執行命令之工作的應用程式。 「伺服器」應用程式是負責使用者介面的應用程式，而且是背景工作角色，代表用戶端將輸入和輸出轉譯成標準格式。

## <a name="console-subsystem"></a>主控台子系統

這是一個全面的詞彙，代表影響主控台和命令列操作的所有模組。 它特別指的是可攜式可執行檔標頭一部分的旗標，此旗標會指定啟動應用程式是否 (命令列/主控台應用程式，而且必須具有啟動) 或 windows 應用程式 (的標準控制碼，而且不需要) 。

主控台主機、命令列用戶端應用程式、主控台驅動程式、主控台 API 介面、pseudoconsole 基礎結構、終端機、設定屬性工作表、進程載入器內的機制和存根，以及與這些形式的應用程式運作相關的任何公用程式都會被視為屬於此群組。

## <a name="console-host"></a>主控台主機

Windows 主控台主機或都 `conhost.exe` 是適用于所有 Windows 主控台 api 的伺服器應用程式，以及使用命令列應用程式的傳統 Windows 使用者介面。 此二進位檔的完整內容（即 API 伺服器和 UI）在過去屬於 Windows `csrss.exe` 、重要的系統進程，而且基於安全性和隔離的目的而分歧。 未來， `conhost.exe` 將會繼續負責進行 API 呼叫服務和轉譯，但使用者介面元件的目的是要透過 pseudoconsole 將其委派給終端機。

## <a name="pseudoconsole"></a>Pseudoconsole

這是從其他平臺 pseudoterminal 或「PTY」的 Windows 模擬。 它會嘗試比對 PTYs 的一般介面原理，提供簡單的雙向通道來進行文字通訊，但它會在具有大型相容性層的 Windows 上進行補充，以將此設計原理之前所撰寫的 Windows 應用程式，從傳統主控台 API 介面變更為簡單的文字通道通訊形式。 終端機可以使用 pseudoconsole 從主控台主機取得使用者介面元素的擁有權， `conhost.exe` 同時讓它負責 API 服務、轉譯和相容性工作。

## <a name="terminal"></a>終端

終端機是命令列應用程式的使用者介面和互動模組。 現在，它是一種軟體，用來表示在具有顯示器監視器、鍵盤和雙向序列通道的實體裝置之前的用途。 它負責以各種形式來收集使用者的輸入、將它轉譯和編碼，以及將任何特殊的命令資訊轉譯成單一文字資料流程，並將其提交至 PTY，以在 `STDIN` 命令列用戶端應用程式的通道上傳輸。 它也負責透過 PTY 接收來自用戶端應用程式通道的資訊，將 `STDOUT` 任何特殊資訊解碼、配置所有文字和其他命令，並以圖形方式呈現給使用者。
